# TFL Arrivals Take Home

## About
The purpose of this test is for us to assess how you analyse requirements, solve problems, approach UX,
use framework and language features appropriately, structure code and verify your solution works correctly.

## Task
*Display all arrivals for Tower Hill station using TFL's stop point arrivals API.*

TFL [provides an API for upcoming arrivals at a given stop point](https://api-portal.tfl.gov.uk/api-details#api=ReleasedUnifiedAPIProd&operation=StopPoint_Arrivals)
(underground station in this case). These arrivals span multiple lines, platforms and destinations. In the case of
Tower Hill, this spans the Circle and District lines.

We have provided an implementation of `TflApiService` with the `getStopPointArrivals` method and
`TOWER_HILL_UNDERGROUND_NAPTAN_ID`. We have set up the solution with Angular 11, Jest and NgRx however you are free
to use any libraries you feel appropriate. Alternatively, a solution in React is also acceptable. 

To use the TFL API, you will need to replace the `APP_KEY` and `APP_ID` variables with your own by
[registering for the TFL API (free)](https://api-portal.tfl.gov.uk/signup). An example response is also provided below.

**For this task we want to:**
1. Display the arrivals, grouped by the platform direction (eg. Eastbound or Westbound) in order of next to arrive.
2. Ensure the information displayed is always up to date.

**Extensions:**
1. Provide an option to display the next 3 arrivals by time.

**The time spent on the task should not exceed 4 hours.**
Please check in your solution in this repository.
